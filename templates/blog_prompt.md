## 규칙
1. 위 어투 가이드와 발췌 예시의 톤을 정확히 따를 것
2. 반말체(87.5%) 위주, 가끔 해요체 혼용
3. 이모지 사용하지 않음
4. 한 문장 평균 34자 내외
5. 사용자의 주관적 경험/의견을 자연스럽게 녹여낼 것
6. 이미지는 관련 텍스트 뒤에 배치
7. **한 줄에 한 문장씩** `\n`으로 구분 (줄당 평균 50자 이하)
8. 텍스트 블록 합산 글자수가 **1200자 미만이면 내용을 보충**할 것

## 글 구조 (맛집 기준 - 6파트 필수)

### 파트 1: 가게이름 (text) → 인용구 버티컬 라인
- 식당 이름만 (소개 문장 없이)
- 에디터에서 자동으로 인용구(좌측 세로 바) 스타일로 렌더링됨

### 파트 2: 영업정보 (text) → bullet(•) 리스트
- **반드시 "영업시간"** 키워드 포함
- 영업시간 / 휴무 / 주차 정보
- 에디터에서 자동으로 각 줄 앞에 • 문자 추가됨
- 정보를 모르면 웹 검색으로 보충

### 파트 3: 구분선 (separator)
- `{"type": "separator"}` 블록 삽입

### 파트 4: 본문 (text + image 교차)
- 가게 분위기, 메뉴 소개, 음식별 상세 리뷰
- 텍스트와 이미지를 교차 배치
- [이미지: 외관/입구]
- 가게 분위기
- [이미지: 내부]
- 메뉴 소개 + 가격
- [이미지: 메뉴판]
- 음식별 상세 리뷰 (맛/식감/비주얼)
- [이미지: 음식 사진들 - 메뉴별로 배치]
- 조합 추천
- [이미지: 조합 사진]

### 파트 5: 구분선 (separator)
- `{"type": "separator"}` 블록 삽입

### 파트 6: 최종평가 (text)
- 자연스러운 문장으로 총평 + 가성비 + 재방문 의사
- **라벨 붙이지 말 것** ("총평:", "가성비:", "재방문:" 같은 접두사 없이)
- 반드시 "재방문" 또는 "또 가" 키워드 포함

### 파트 7: 장소 위젯 (place 필드)
- place 필드로 지정, 에디터에서 맨 마지막에 네이버 지도 위젯 자동 삽입

## 출력 형식 (blocks JSON)
```json
{
  "title": "[가게이름 지역] SEO 키워드 포함 제목",
  "category": "카테고리명",
  "tags": ["태그1", "태그2", ...최소 8개],
  "thumbnail": "/절대경로/대표이미지.jpeg",
  "place": "가게이름 지역",
  "blocks": [
    {"type": "text", "content": "가게이름"},
    {"type": "text", "content": "영업시간 11:00 - 21:00\n휴무 매주 월요일\n주차 불가"},
    {"type": "separator"},
    {"type": "text", "content": "본문 텍스트..."},
    {"type": "image", "paths": ["/절대경로/img1.jpeg"]},
    {"type": "text", "content": "다음 텍스트..."},
    {"type": "image", "paths": ["/절대경로/img2.jpeg", "/절대경로/img3.jpeg"]},
    {"type": "separator"},
    {"type": "text", "content": "전반적으로 만족스러운 곳이다\n가격 대비 양이 넉넉해서 좋았다\n기회 되면 또 가고 싶다"}
  ]
}
```

### blocks 작성 규칙
- 텍스트 블록: 2~6줄, 하나의 소주제, **한 줄에 한 문장**
- 이미지 블록: 1~3장, 직전 텍스트와 관련된 사진
- 구분선 블록: `{"type": "separator"}` - 영업정보 뒤, 최종평가 앞에 반드시 1개씩
- 이미지가 없는 글은 text 블록만으로 구성 (separator는 유지)
- 태그: 8~12개 (가게이름, 지역+맛집, 메뉴명, 지역+카테고리 등)

### 블록 순서 검증 패턴
text → text → **separator** → (text/image 교차) → **separator** → text
