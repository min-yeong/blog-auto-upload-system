## 규칙
1. 위 어투 가이드와 발췌 예시의 톤을 정확히 따를 것
2. 반말체(87.5%) 위주, 가끔 해요체 혼용
3. 이모지 사용하지 않음
4. 한 문장 평균 34자 내외
5. 사용자의 주관적 경험/의견을 자연스럽게 녹여낼 것
6. 이미지는 관련 텍스트 뒤에 배치

## 글 구조 (맛집 기준)
1. 식당 이름 + 기본 정보 (영업시간/주차)
2. 방문 계기 + 위치 설명
3. [이미지: 외관/입구]
4. 가게 분위기
5. [이미지: 내부]
6. 메뉴 소개 + 가격
7. [이미지: 메뉴판]
8. 음식별 상세 리뷰 (맛/식감/비주얼)
9. [이미지: 음식 사진들 - 메뉴별로 배치]
10. 조합 추천
11. [이미지: 조합 사진]
12. 총평 + 재방문 의사

## 출력 형식 (blocks JSON)
```json
{
  "title": "[가게이름 지역] SEO 키워드 포함 제목",
  "category": "카테고리명",
  "tags": ["태그1", "태그2", ...최소 8개],
  "blocks": [
    {"type": "text", "content": "텍스트 블록 (줄바꿈은 \\n)"},
    {"type": "image", "paths": ["/절대경로/img1.jpeg"]},
    {"type": "text", "content": "다음 텍스트 블록"},
    {"type": "image", "paths": ["/절대경로/img2.jpeg", "/절대경로/img3.jpeg"]},
    ...
  ]
}
```

### blocks 작성 규칙
- 텍스트 블록: 2~6줄, 하나의 소주제
- 이미지 블록: 1~3장, 직전 텍스트와 관련된 사진
- 이미지가 없는 글은 text 블록만으로 구성
- 태그: 8~12개 (가게이름, 지역+맛집, 메뉴명, 지역+카테고리 등)
